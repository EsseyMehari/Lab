1. Testing Strategy:

a. Creation and Destruction:
- Created a hash table using ht_create() and verified it was not NULL.
- Called ht_free() to ensure no crashes occurred.
- Repeated creation and destruction multiple times.

b. Insertion Tests:
- Inserted multiple key-value pairs into the hash table.
- Inserted duplicate keys to confirm values were updated correctly.
- Inserted many elements to trigger resizing of the table.

c. Lookup Tests:
- Looked up keys that exist and verified correct values were returned.
- Looked up keys that do not exist (expected NULL).
- Performed lookups after resizing to ensure data was still correct.

d. Removal Tests:
- Removed existing keys and confirmed they were no longer found.
- Attempted to remove keys that do not exist (no effect expected).
- Tested removal followed by insertion to ensure table still worked.

e. Collision Handling:
- Inserted multiple keys that likely hash to similar positions.
- Verified that linear probing correctly resolved collisions.
- Ensured lookup still worked after collisions occurred.

f. Resizing:
- Inserted enough elements to exceed load factor.
- Verified table resized correctly.
- Confirmed all previous elements were still accessible after resizing.

g. pcode Program Tests:
- Ran program with provided input files (postalcodes_20.in, postalcodes_100.in).
- Verified multiple postal codes for the same city were stored and printed correctly.
- Checked that order of postal codes matched input file.

h. Edge Cases:
- Tested with empty input file.
- Tested lookup for cities not in the dataset.
- Tested repeated input of same city from stdin.

2. Automated Testing:
- Compiled using make and tested program execution.
- Used sample input files and custom small test files.
- Compared output manually with expected results.

3. Tools Used:
- GCC compiler with -Wall -Wextra flags.
- VS Code terminal for running and testing.
- (Optional) Valgrind to check for memory leaks.

4. Results:
- All functions behaved as expected.
- Hash table handled collisions using linear probing.
- Resizing preserved all data correctly.
- Program produced correct output for all tested inputs.
